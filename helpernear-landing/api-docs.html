HelperNear API Reference

Complete documentation for all backend endpoints. Built with NestJS, PostgreSQL, Prisma, and JWT authentication.

BASE URL http://localhost:3000/api/v1
Authentication Guide
How the frontend should handle authentication.
1. Call send-otp → 2. User enters OTP → 3. Call verify-otp → 4. Store tokens → 5. Add Bearer header to all requests

How to attach token to every request

After login, store the accessToken and add it to every protected API call:

// Store tokens after login
localStorage.setItem('accessToken', response.data.accessToken);
localStorage.setItem('refreshToken', response.data.refreshToken);

// Add to every API call
const token = localStorage.getItem('accessToken');
fetch('/api/v1/users/me', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});

// If you get 401, refresh the token
const res = await fetch('/api/v1/auth/refresh', {
  method: 'POST',
  body: JSON.stringify({ refreshToken: localStorage.getItem('refreshToken') })
});
const { data } = await res.json();
localStorage.setItem('accessToken', data.accessToken);
Auth
Phone-based OTP authentication. No passwords needed.
POST /auth/send-otp Send OTP to phone number

Sends a 6-digit OTP to the user's phone via Twilio SMS. In dev mode, OTP is logged to the server console. Rate limited to 3 requests per 10 minutes.

Request Body
Response
Code Example
FieldTypeRequiredDescription
phonestringRequiredPhone in E.164 format e.g. +919876543210

200 OTP sent successfully

{ "success": true, "message": "OTP sent successfully", "data": { "message": "OTP sent successfully" } }

403 Too many requests

const res = await fetch('/api/v1/auth/send-otp', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ phone: '+919876543210' })
});
const data = await res.json();
POST /auth/verify-otp Verify OTP → get JWT tokens

Verifies the OTP and returns access + refresh tokens. Creates a new user account if phone number is new. OTP expires in 5 minutes and can only be used once.

Request Body
Response
Code Example
FieldTypeRequiredDescription
phonestringRequiredSame phone used in send-otp
otpstringRequired6-digit OTP received via SMS

200 Login successful

{
  "success": true,
  "message": "Login successful",
  "data": {
    "user": { "id": "uuid", "phone": "+919876543210", "role": "CUSTOMER", "isActive": true },
    "accessToken": "eyJhbGci...",
    "refreshToken": "eyJhbGci..."
  }
}

400 Invalid or expired OTP

const res = await fetch('/api/v1/auth/verify-otp', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ phone: '+919876543210', otp: '123456' })
});
const { data } = await res.json();
// Save tokens
localStorage.setItem('accessToken', data.accessToken);
localStorage.setItem('refreshToken', data.refreshToken);
POST /auth/refresh Refresh access token

Access tokens expire in 15 minutes. Call this endpoint with the refresh token to get a new access token without making the user login again. Refresh tokens are valid for 7 days.

Request Body
Response
FieldTypeRequiredDescription
refreshTokenstringRequiredThe refresh token from verify-otp response
{ "data": { "accessToken": "eyJ...", "refreshToken": "eyJ..." } }
POST /auth/logout Logout — requires Bearer token
🔒 Protected endpointRequires Authorization: Bearer <accessToken> header.

Logs out the current user. Frontend must delete stored tokens after calling this.

// After calling logout, clear tokens
localStorage.removeItem('accessToken');
localStorage.removeItem('refreshToken');

Categories
Service categories like Plumber, Electrician, Maid. GET is public, POST/PATCH/DELETE require Admin role.
GET /categories List all categories — Public

Returns all available service categories. No auth required. Use this to populate category dropdowns in the app.

[

  { "id": "uuid", "name": "Electrician", "icon": "⚡" }
]

🔒 Admin onlyRequires Bearer token with ADMIN role.
FieldTypeRequiredDescription

GET /workers/nearby Search nearby workers — Core feature

Returns workers sorted by distance within the given radius. This is the core geo search feature of HelperNear.
